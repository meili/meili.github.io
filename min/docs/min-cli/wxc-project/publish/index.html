发布组件

Min组件库项目 下的每个组件是独立完整的 package，用户可以将 组件package 发布到 NPM 仓库

$ min publish

为什么使用，她与 npm publish 有什么区别

  • 支持批量发布
  • 基于语义化版本控制规范（SemVer）
  • 自动更新version版本号
  • 自动打标签并推送tag到托管服务器
  • 全自动化流程操作，轻松管理组件版本发布

发布步骤

  • 进入Min组件库项目的根目录
$ cd ~/your_weapp_project/
  • 发布组件，以 MinUI 组件库的 toast 组件举例
$ min publish toast

  • 此时，进入发布流程的选项界面，开发者按照命令行上的提示输入或选择来完成操作

    • 选择组件新版本号。（假设当前 toast 组件版本号为1.0.0，Min 工具将会基于语义化版本控制规范 SemVer 输出以下可选版本列表）
    - Patch (1.0.1)
- Minor (1.1.0)
- Major (2.0.0)
- Prepatch (1.0.1-0)
- Preminor (1.1.0-0)
- Premajor (2.0.0-0)
- Prerelease
- Custom
    • 开发者选择新版本号后，会看到以下确认信息。（假设新版号选择 Patch (1.0.1)）
    Changes:
- wxc-toast: 1.0.0 => 1.0.1

Are you sure you want to publish the above changes? (ynH)
    • 确认是否发布
    - 输入 y 确认发布
- 输入 n 退出发布
- 输入 h 显示帮助
  • 完成上个步骤，Min 工具将进入自动化发布阶段
    • 更新组件 version 版本号
    • 发布 组件package 到 NPM 仓库
    • 生成 tag @minui/wxc-toast@1.0.1 并推到 git 服务器

批量发布

  • 开发者也可以直接执行指令，用于选择单个或批量发布方式
$ min publish

  • 此时，进入选择发布方式 的选项界面，开发者按照命令行上的提示输入或选择来完成操作

    • 请选择发布方式，此时请选择 发布项目里的每个组件 
    - 手动选择要发布的组件
- 发布项目里的每个组件

  • 选择 发布项目里的每个组件 ，将进入【使用姿势 - 发布流程】环节

  • 选择 手动选择要发布的组件 ，与 min publish [name] 用法相同，都是用于发布单个组件，此时 Min 工具将会输出组件列表，开发者从列表里选择需要发布的后，再进入【使用姿势 - 发布流程】环节

注意：

在发布组件前，开发者需要确认当前Min组件库项目里是否具备以下几点

  • 要求 packages 组件库目录下里至少有一个 组件package
  • 要求当前项目已使用 git 源代码管理，且代码已提交
