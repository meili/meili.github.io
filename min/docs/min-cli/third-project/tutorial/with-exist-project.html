已有项目中使用 Min

本节介绍在已有项目中使用 Min，即在已有微信小程序项目中使用 MinUI 组件。准备工作如下：

1、安装 Min

$ min install -g @mindev/min-cli

2、了解 MinUI 组件库

安装 MinUI 组件

下面开始介绍在已有项目中使用 Min，主要是在已有小程序项目中使用 MinUI 组件，此处以 MinUI 组件库的 wxc-toast 组件为例。

1、已有小程序示例项目

这里我用在微信开发者工具中初始化的一个简易的微信小程序为示例项目，小程序的目录结构如下：

- wx-init  // 项目名
    - pages // 页面目录
        - index // index 页面
            - index.js
            - index.json
            - index.wxml
            - index.wxss
    - utils // 工具目录
        - util.js
    - app.js
    - app.json
    - app.wxss

2、添加package.json文件。

Min 安装组件，是执行 npm install 再从 node_modules 中把组件编译到组件的安装目录中。 如果你的项目中还没有 package.json 文件，则执行 npm install 会报错，所以必须先在项目中添加 package.json 文件。如果不了解package.json的基本配置，可直接复制下面的配置并填写好相关信息：

{
  "name": "wxtest",
  "description": "wx test project",
  "version": "1.0.0",
  "repository": {
    "type": "git",
    "url": ""
  },
  "author": "",
  "license": "MIT",
  "dependencies": {}
}

2、安装 wxc-toast 组件：

$ min install @minui/wxc-toast

执行命令后，会提示 请设置安装路径 (dist/packages)，默认的路径为 dist/packages，可输入路径也可直接回车使用默认路径。这里输入路径： packages，将组件安装在根目录的 packages 目录下，min 会自动创建该目录。

MinUI 组件的安装路径应该也设置在微信开发者工具中浏览目录的某个路径下，如果设置在浏览目录以外的路径中，在微信开发者工具中会访问不到组件。

安装 wxc-toast 组件后，在项目的根目录下会生成一个 Min 的配置文件： min.config.json ，其配置如下所示：

{
  "npm": {
    "dest": "packages"
  }
}

npm.dest 即为组件编译后的保存路径。之后再安装组件都会保存在这个目录中。

3、在页面中使用 wxc-toast。

这里在 index 页面中使用 wxc-toast 组件。在 index.json 中配置 usingComponents 字段，wxc-toast 组件的路径是 index 页面相对于 wxc-toast 组件的路径。使用wxc-toast组件的示例如下：

index.json 文件：

{
  "usingComponents": {
    "wxc-toast": "../../packages/@minui/wxc-toast/dist/index"
  }
}

index.js 文件：

Page({
  data: {},
  onLoad: function () {},
  showToast() {
    let $toast = this.selectComponent(".J_toast")
    $toast && $toast.show()
  }
})

index.wxml 文件

<view>
  <wxc-toast
          class="J_toast"
          text="Hello World"></wxc-toast>
  <button bindtap="showToast">调用 show() 方法显示</button>
</view>
