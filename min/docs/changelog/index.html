Min Cli 版本日志

v1.0.5（2017.12.11）

  • A 新增 组件支持导出多个入口
  • A 新增 使用 min install/update/build 命令时增加 “是否启用 es6 转 es5” 选项
  • A 新增 使用 min build 命令编译依赖组件，增加“请设置编译后的保存路径” 选项
  • F 修复 在 less 里使用 “运算符” 转换问题
  • F 修复 在 less 里使用 “@import” 找不到文件错误

v1.0.4（2017.11.29）

  • A 新增 支持es6/es7新特性
  • A 新增 支持创建空scope名称的组件
  • F 修复 引用wxc组件路径依赖分析错误问题
  • F 优化 脚手架首页模板换成@minui/wxc-example-menu导航菜单组件
  • F 优化 min init 初始化项目，规范组件安装路径、交互式问答输入验证
  • F 优化 min new 创建组件的规则校验，只能包括小写字母，支持“-”分隔
  • F 优化 config配置，包括scope和prefix

v1.0.3（2017.11.16）

  • F 修复windows系统下wxa文件的全局模板编辑后没有重新编译所有页面问题
  • F 修复import第三方npm包解析找不到模块问题
  • F 修复标签属性转为小写问题

v1.0.2（2017.11.10）

  • F 修复 windows 系统兼容问题（min init 初始化项目安装失败；min dev 生成页面路径错误；其他问题）
  • A 增加 min install | update 支持自定义设置安装后的保存路径
  • F 修复 tabbar 的 iconPath 路径强制依赖问题
  • F 修复 package 组件的 README.md 文件编辑后没有实时编译示例页面的文档
  • F 修复 min xxx -h 没有打印 help 信息
  • F 优化 min dev 调试多个页面命令行参数使用英文逗号分隔，暂不支持空格分隔，min install 和 min update存在同样的问题
  • F 修复 min update 命令行缺少参数报错

v1.0.1（2017.11.02）

  • A 初始版本
