  • 创建页面：

    $ min new demo

    执行上述命令，会提示 请设置新页面的中文标题，输入页面标题并按下回车键，min 会在 src/pages 目录下新建 demo 文件夹，demo 文件夹中有index.wxp文件，wxp 是 min 的单文件页面开发模式。新建页面后，min 还会执行 min build，编译项目。

    注意，新建 demo 页面后，还需配置 demo 页面的入口，比如在 home 页面中做一个跳转链接到 demo 页面中。

    一个wxp文件分为三个部分，template 代码块，该区块中写 wxml，script 代码块，style 代码块。wxp页面的代码示例如下。值得注意的是，在 script 代码块中，config 字段就是小程序的页面配置项，相当于 index.json 文件。单文件模式极大的简化了微信小程序的多文件开发方式，是 Min 的一大特性。

    <template>
  <view>Demo Page</view>
</template>

<script>
export default {
  config: {
    navigationBarTitleText: 'demo页面',
    usingComponents: {}
  },
  data: {}
}
</script>

  • 安装组件，以安装 minui 组件库中的 wxc-toast 组件为例：

    $ min install @minui/wxc-toast

  • 使用组件。在 demo 页面中使用 wxc-toast 组件，首先应在 config 中的 usingComponents 字段中配置自定义组件的组件名及组件路径，然后在template中使用该组件标签。一个完整使用示例如下：

    <!--src/pages/demo/index.wxp -->
<template>
  <view>
    <wxc-toast
            class="J_toast"
            text="Hello World"></wxc-toast>
    <button bindtap="showToast">调用 show() 方法显示</button>
  </view>
</template>

<script>
export default {
  config: {
    navigationBarTitleText: 'demo页面',
    usingComponents: {
      'wxc-toast': '@minui/wxc-toast'
    }
  },
  data: {},
  showToast() {
    let $toast = this.selectComponent(".J_toast")
    $toast && $toast.show()
  }
}
</script>

</style>
