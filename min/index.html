Min 的特性和进阶介绍

为什么要选择Min

NPM

支持使用第三方 NPM 资源，自动处理 NPM 资源之间的依赖关系，完美兼容所有无平台依赖的 NPM 资源包

单文件模式

单文件的方式开发方式，让目录结构更清晰，开发更方便

ES2015

可使用 Generator Fu-nction / Class / Async Function 等特性，大大提升开发效率

Promise

通过 polyfill 让小程序完美支持 Promise，解决回调烦恼

编译器

支持样式编译器：Less/Postcss，代码编译器：Babel

自动化集成文档

自动集成组件的 Demo示例 和 API文档，生成小程序示例应用，便于分享组件和真机体验

全局公共模版

支持页面模板管理，自动下发到每个页面，极大的减少维护成本

全局可配置化

支持CSS样式全局可配置化，解决不同小程序间的差异化

完善的开发流程

从组件开发环境，组件的新建、到组件的调试和发布、以及组件文档的自动集成，大大提升组件开发效率

丰富的 UI 组件库

基于规范的 MinUI 组件库，帮助用户快速搭建一个小程序应用

Min 灵活多变

既能开发组件库也能开发小程序应用，兼容时下热门的小程序组件开发框架，为其舔砖加瓦、赋予新能量
